Echarts的介绍

Echarts是一个使用JavaScript实现的开源可视化库，兼容性强，底层依赖矢量图形库ZRender，提供直观，交互丰富，可高度个性化定制的数据可视化图表。官网：https://echarts.apache.org/zh/index.html

特点：

  1. 开源免费
  2. 兼容性强
  3. 丰富的可视化类型
  4. 多种数据格式支持
    • key-value数据格式
    • 二维表
    • typedArray格式
  5. 流数据的支持
    • 流数据的动态渲染(需要哪些数据就加载哪些数据)
    • 增量渲染技术(哪些数据变化，只重新渲染变化的数据)

Echarts的基本使用

  1. 步骤一：引入echarts.js文件
  2. 步骤二：准备一个呈现图表的盒子
  3. 步骤三：初始化echarts实例对象
  4. 步骤四：准备配置项
  5. 步骤五：将配置项设置给echarts实例对象

相关配置项

  • xAxis：直角坐标系中的x轴
  • yAxis：直角坐标系中的y轴
  • series：系列列表。每个系列通过type决定自己的图表类型

通用配置

通用配置指的就是任何图表都能使用的配置

  • 标题：title
    • 文字样式 textstyle
    • 标题边框 borderWidth borderColor borderRedius
    • 标题位置 letf、top、right、bottom
  • 提示：tooltip
    • 触发类型：trigger
      • item
      • axis
    • 触发时机：triggerOn
      • mouseover
      • click
    • 格式化提示框内容：formatter
      • 字符串模板
      • 回调函数
  • 工具栏按钮：toolbox
    • 内置有导出图片、数据视图、动态类型切换、数据区域缩放、重置五个工具
  • 图例：legend
    • 用于筛选系列,需要和series配合使用
    • legend中的data是一个数组
    • legend中的data的值需要和series数组中某组数据的name值一致

直角坐标系的常用配置

  • 配置1：网格 grid
  • 配置2：坐标轴 axis
  • 配置3：区域缩放 dataZoom

配置1：网格 grid

grid是用来控制直角坐标系的布局和大小

x轴和y轴就是在grid的基础上继续宁绘制的

  • 显示grid

配置2：坐标轴 axis

  • 坐标轴类型type

    value：数值轴，自动会从目标数据中读取数据

    category：类目轴，该类型必须通过data设置类目数据

  • 显示位置 position

    xAxis：可取值为top或者bottom

    yAxis：可取值为left或者right

配置3：区域缩放 dataZoom

dataZoom用于区域缩放，对数据范围过滤，x轴和y轴都可以拥有

dataZoom是一个数值，意味着可以配置多个区域缩放器

  • 类型type
    • slider：滑块
    • inside：内置，依靠鼠标滚轮或者双指缩放
  • 指明产生的作用的轴
    • xAxisIndex：设置缩放组件控制的是哪个x轴，一般写0即可
    • yAxisIndex：设置缩放组件控制的是哪个y轴，一般写0即可
  • 指明初始状态的值

一、柱状图

柱状图示例：

示例代码：

常见效果

  • 标记：最大值 最小值 平均值
    • markPoint（最大值，最小值）
    • markLine（平均值）
  • 显示：数值显示 柱宽度 横向柱状图
    • label（数值显示）
    • barWidth（柱宽度）
    • 更改x轴和y轴的角色（横向柱状图）

柱状图的特点

  • 柱状图描述的是分类数据，呈现的是每一个分类中有多少，通过柱状图，可以很清晰的看出每个分类数据的排名情况。

二、折线图

折线图的绘制基本上跟柱状图一直，不同的是type:'line'

常见效果：

  • 标记：最大值 最小值 平均值 标注区间
    • markArea配置标注区间
  • 线条控制：平滑 风格
    • smooth、lineStyle
  • 填充风格：
    • areaStyle:{}
  • 紧挨边缘：
    • boundaryGap
  • 缩放：脱离0值比例
    • scale
  • 堆叠图：
    • stack

折线图的特点

  • 折线图常用来分析数据随时间的变化趋势

三、散点图

散点图可以帮助我们推断出变量间的相关性（比如身高和体重的关系）

  • x轴数据和y轴的数据需要使用一个二维数组（比如[[身高1]体重1,[身高2，体重2]]）
  • 图表类型：
    • 在series下设置type:scatter
    • xAxis和yAxis的type都要设置为value
  • 调整：
  • 将坐标轴都设置为脱离0值比例，scale

常见效果：

  1. 气泡图
    1. 散点的大小不同 symbolSize
    2. 散点的颜色不同 itemStyle.color
  2. 涟漪动画效果
    1. type:effectScatter
    2. showEffectOn:''
    3. rippleEffect:{scale:10}

散点图的特点

  • 散点图可以帮助我们推断出不同纬度数据之间的相关性
  • 散点图也经常用在地图的标注上

交互的API

  • 全局的echarts对象
    • 全局的echarts对象是引入echarts.js文件之后就可以直接使用的对象
  • echartsInstance对象
    • eChartsInstance对象是通过echarts.init方法调用之后得到的

全局API方法：

  1. init
    • 初始化echarts实例对象
    • 使用主题
  2. registerTheme
    • 注册主题
  3. registerMap
    • 注册地图数据
    • geo组件使用地图数据
  4. connect
    • 可以将多个图表进行关联，传入联动目标为ECharts实例对象，支持数组。

echartsInstance对象的API：

  1. setOption方法

    • 设置或修改图表实例的配置项以及数据
    • 多次调用setOption方法，合并新旧配置

  2. resize方法

    • 重新计算和绘制图表

    • 一般和window对象的resize事件结合使用

  3. on\off方法

    • 绑定或者解绑事件处理函数
      • 鼠标事件
      • ECharts事件

  4. dispatchAction方法

    • 触发某些行为

    • 使用代码模拟用户的行为

  5. clear方法

    • 清空当前实例，会移除实例中所有的组件和图表
    • 清空后可以再次setOption将图表展示出来

  6. dispose方法

    • 销毁实例
    • 销毁后实例无法再被使用
